Curriculum vitae

I wonder what the point of a CV is. If I’m not doing the conventional academic thing, I don’t really need the typical CV. But why is it desireable? To some degree, it’s a summary of a person. My website is also a summary of me too, but it ignores some typical contents of a CV.

I think a CV is intended to make someone sound impressive. I sort of don’t like doing that, but I’m curious about what it would sound like. And I’m applying to a graduate school (April 1, 2014). So here I go.

If you dare read this, keep in mind that I’m inflating things in order to play the real-world game.

Notable software

I’ve written quite a lot of software. Here are some free software projects that a lot of people seem to use.

  • OpenPrism has become the defacto standard site for searching for United States government data.
  • DumpTruck is a light, Python-style interface to SQLite3.
  • Urchin is a test framework for shell.
  • My website’s source code is quite popular.
  • special_snowflake detects unique keys in spreadsheets.
  • pickle-warehouse acts like a Python dictionary, but it saves things to disk.
  • craigsgenerator retrieves listings from craigslist in a Pythonic way.
  • treasury.io provides daily data on the flows of money through the United States treasury.
  • Scott is a system for obtaining and collaboratively annotating information about the dredging and filling of wetlands.
  • ScraperWiki provides tools for acquiring and analyzing data.
  • South Africa Map of Financial Inclusion
  • The NYC Crime Map Data API scraper collects and reformats data released by the New York City Police Department.

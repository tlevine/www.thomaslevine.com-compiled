What happened? I recently updated Ruby and the various gems related to my nanoc installation, and this apparently changed the behavior of haml.

Haml pretty-prints the html that it produces, but it has a command for preserving whitespace in tags like pre and textarea where you care about the whitespace. That command is ~, as opposed to =, and it just runs the function find_and_preserve with the default tags.

I was using =, so the whitespace in the code blocks was not being preserved. I fixed this by changing a = to a ~.
